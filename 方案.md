# CodeCard — 完整方案

## 概述

一款 Android 手机 APP（React Native），帮助用户系统学习编程知识。首批上 C++，后续扩展数据结构、计算机组成原理、计算机网络、操作系统等学科。

核心设计理念：**像刷短视频一样学编程** — 卡片滑动、即时反馈。

## 架构核心原则

### 扩展性

**新增一个学科只需两步：**

```
1. src/data/courses/ 下新建学科文件夹 + 数据文件
2. src/data/courses/index.ts 里加一行注册
```

**不需要改动的文件：** 导航、Screen、Store、组件全部通用，不 import 任何具体学科的代码。

### 数据隔离

每个学科对应一个文件夹，学科间互不引用：

```
src/data/courses/
├── index.ts              ← 所有学科注册入口（一行一个学科）
└── cpp/                    ← C++ 专属文件夹
│   ├── index.ts            ← C++ 课程组装
│   ├── 01-basics/index.ts
│   ├── 02-advanced/index.ts
│   ├── 03-oop/index.ts
│   ├── 04-stl/index.ts
│   ├── 05-generics/index.ts
│   └── 06-modern/index.ts
├── data-structures/        ← 预留
├── computer-org/           ← 预留
└── networks/               ← 预留
```

### 通用 vs 学科专属

| 类别 | 位置 | 说明 |
|------|------|------|
| 类型定义 | `src/types/` | 所有学科共用 |
| 导航 | `src/navigation/` | 不涉及任何具体学科 |
| Screen | `src/screens/` | 用 courseId/nodeId 泛查找数据 |
| 通用组件 | `src/components/cards/` | 根据 cardType 渲染 |
| 动画组件 | `src/components/animations/` | 跨学科复用 |
| Store | `src/store/` | 进度/XP/打卡 通用 |
| C++ 课程数据 | `src/data/courses/cpp/` | C++ 专属，只 export 纯数据 |
| 学科注册 | `src/data/courses/index.ts` | 学科列表 |

## 用户动线

```
[Tab: 学习]
     │
     ▼
HomeScreen（主页）
     │
     ├── 顶部：学科选择器（从 courses[] 遍历，默认选中第一个）
     │
     ├── 中部：当前学科的知识模块列表（按章节分组）
     │    └── 点击模块 → navigate('Node', { courseId, nodeId })
     │
     └── 底部：「继续学习」浮动按钮
           │
           ▼
NodeScreen（卡片翻页）
     │
     ├── 从 courses 中泛查找：courseId → course.nodes → nodeId → node.cards
     │
     ├── 根据 cardType 分发渲染：
     │    ├── concept   → ConceptCard
     │    ├── code      → CodeCard
     │    ├── animation → AnimationCard（嵌入动画组件）
     │    └── practice  → PracticeCard
     │
     ├── 翻页按钮，完成时更新 store.completedCards
     │
     └── 返回后，学习路径节点状态更新

[Tab: 进度]  → ProgressScreen（热力图 + 等级 + 徽章）

[Tab: 设置]  → SettingsScreen（用户切换、重置、关于）
```

## 数据模型

### 三层结构：Course → PathNode → Card

```typescript
// ===== 学科层 =====
interface Course {
  id: string;
  title: string;
  icon: string;
  color: string;
  nodes: PathNode[];
}

// ===== 学习路径节点 =====
interface PathNode {
  id: string;
  courseId: string;
  type: 'knowledge' | 'quiz';
  chapter: string;
  title: string;
  cards: Card[];
}

// ===== 卡片（最小学习单元）=====
interface Card {
  id: string;
  cardType: 'concept' | 'code' | 'animation' | 'practice';
  content: TextContent | CodeContent | AnimationContent | PracticeContent;
}
```

### 进度模型

```typescript
interface ProgressState {
  profiles: Record<string, UserProfile>;
  activeProfile: string;
  global: GlobalProgress;
}

interface UserProfile {
  name: string;
  courses: Record<string, CourseProgress>;
}

interface CourseProgress {
  completedCards: string[];
  currentNodeId: string;
  xp: number;
  quizScores: Record<string, number>;
}
```

## 课程数据规范

每个学科文件夹下的文件 **只 export 纯数据**，不含业务逻辑：

```typescript
// src/data/courses/cpp/01-basics/index.ts
import type { PathNode } from '../../../types';

export const basicsNodes: PathNode[] = [
  {
    id: 'cpp-01-start',
    courseId: 'cpp',
    type: 'knowledge',
    chapter: '01 起步',
    title: '第一个程序',
    cards: [
      {
        id: 'cpp-01-start-c1',
        cardType: 'concept',
        content: {
          title: 'Hello World 是什么',
          body: '每个 C++ 程序都从 main 函数开始...',
        },
      },
      {
        id: 'cpp-01-start-c2',
        cardType: 'code',
        content: {
          title: '第一个程序',
          code: '#include <iostream>\n\nint main() {\n  std::cout << "Hello World";\n  return 0;\n}',
          language: 'cpp',
          highlights: [0, 2, 3, 4, 5],
        },
      },
    ],
  },
];
```

学科注册入口：

```typescript
// src/data/courses/index.ts
import type { Course } from '../types';
import { cppCourse } from './cpp';

// 新学科在这里加一行
export const courses: Course[] = [cppCourse];
```

## 设计原则

1. **极端简洁** — 启动 30 秒内进入学习，无引导页
2. **一条线走到底** — 学科内线性路径
3. **卡片不文字** — 每屏一个信息单元
4. **做对立即爽** — 绿色 + 震动 + XP 飘字
5. **做错不惩罚** — 不扣 XP，展示正确答案 + 解析
6. **火焰就是命** — 连续打卡是最强留存锚点
7. **代码就是练习** — 不只选择题，看代码/写代码
