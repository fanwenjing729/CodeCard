## AI 开发项目如何申请软著

### 政策变了什么

2026 年 3 月 15 日起，软著审查从"形式审查"升级为"实质审查"：

| 变化 | 之前 | 之后 |
|------|------|------|
| 审查方式 | 看格式对不对 | AI 预审 + 人工复核 |
| AI 代码 | 不查 | 系统自动比对全网代码库 |
| 诚信承诺 | 无 | 必须手写抄录承诺段落 |
| 申请表 | 旧版 | 3.15 启用新版，旧表废止 |
| 功能描述 | ≥300 字 | **500-1300 字**，需详述研发背景、核心架构、技术实现 |
| 驳回率 | 正常 | 暴涨 40%+，涉 AI 项目补正率 +150% |
| 申请周期 | ~1 个月 | 2-5 个月 |

### 核心问题：代码是 AI 写的，还能申请吗？

**纯 AI 生成 → 不行。AI 辅助 + 人工设计主导 → 可以，但必须走合规声明通道。**

2026.5.27 对 CodeCard 代码做了全网比对搜索，8 个核心模式全部零命中：

| 搜索内容 | 结果 |
|---------|------|
| `xpForLevelStart` / `calcLevel` 三角数列 XP 公式 | 0 条 |
| `MemoryBoxStep` / `VarAlloc` / `ScopeCodeStep` 动画类型 | 0 条 |
| `animCodeConditionBg` / `animCodeSkippedBg` / `gridEmptyStroke` 主题令牌 | 0 条 |
| MIGRATIONS 手写版本迁移链 `Record<number, (data: any) => any>` | 类似但不同 |
| `cardType: 'concept' \| 'code' \| 'animation' \| 'practice'` 卡片联合类型 | 0 条 |
| Reanimated MemoryBox 字节偏移动画 | 0 条 |
| "CodeCard" 整体项目名 | 0 条 |
| `AGENTS.md` What NOT to read 访问控制表 | 类似概念但不同 |

关键不是"代码谁写的"，而是"你有没有智力投入"。你的智力投入证据：

| 证据类型 | CodeCard 已有的 |
|----------|:--:|
| 架构设计文档 | `docs/` 下 20+ 份设计文档 |
| commit 历史 | 几十个 commit，持续迭代 |
| 课程内容 | C++ 知识点编排、题目设计（人工创作） |

| 代码原创性 | 全网比对零命中，不是复制粘贴 |

## 申报策略：走 AI 合规声明通道

### 为什么不能走标准通道

新版申请表要求手写抄录承诺段落，内容为：

> "确认并声明本软件确系人的独立开发，**未使用 AI 开发编写代码、撰写文档或生成登记申请材料**"

CodeCard 确实使用了 Claude Code 生成代码，签这句话不实。2026 年新规将 AI 代写明确定义为"非正常申请"，隐瞒 AI 使用一旦被查出 → 驳回 + 列入版权登记失信名单 + 纳入个人征信。

**必须走 AI 辅助申报通道，提交《AI 合规及原创性声明》。**

### AI 合规声明怎么写（5 要素）

| 要素 | CodeCard 的写法 |
|------|----------------|
| ① 软件基本信息 | CodeCard 学习软件 V1.0，版本号、流水号 |
| ② AI 工具说明 | Claude Code（Anthropic 出品），用于代码生成（工具层），所有生成代码经人工审查和修改 |
| ③ 原创性声明 | 核心架构（离线优先 Zustand + 手写迁移链）、动画系统（6 种动画类型体系）、课程内容（C++ 知识点编排与题目设计）、XP 等级公式（三角数列闭环验证）均由人类独立设计 |
| ④ 人工修改比例 | 远超 30% 阈值。类型系统设计、store 架构、动画接口、教学内容、主题令牌均为人工从零设计，AI 仅负责具体代码行的拼写 |
| ⑤ 合规承诺 | 不涉及自行训练模型，AI 生成内容处于人类审查控制下，遵守《生成式人工智能服务管理暂行办法》 |

### "人类主导"三大认定标准 — CodeCard 全部满足

| 标准 | CodeCard 的证据 |
|------|----------------|
| **需求主导**：软件需求、功能定位、核心逻辑由人类确定 | 20+ 份设计文档覆盖架构、动画、课程、store、主题、缩放、暗色模式等所有系统 |
| **实质性修改**：AI 生成代码经人类做核心修改（≥30%） | 类型系统、动画接口、迁移链、主题令牌、课程内容、XP 公式均为人工原始设计 |
| **成果把控**：最终功能、呈现形式由人类决策 | `AGENTS.md` 精确控制 AI 的文件读取范围，证明人在回路、人做决策 |

### 附带证明材料清单

| 材料 | 说明 |
|------|------|
| 架构设计文档索引 | `docs/` 下 26 份设计文档列表，每份标注设计日期和内容摘要 |
| 关键设计决策记录 | 手写迁移链 vs zustand/persist（Fabric 兼容）、动画类型从下往上生长（非顶层设计）、三角数列 XP 与闭式解反函数共存（数学验证） |
| 教学内容原创性说明 | 知识桥接（struct→class）、前向引用（this→运算符重载）、口诀式教学语言（"const 写得多，bug 写得少"） |
| commit 迭代历史 | 几十个 commit 的 log，证明渐进开发而非一键生成 |
| 全网代码比对结果 | 8 个核心模式搜索零命中（2026.5.27 执行） |

## ① 源代码文档制作

### 规则

- 60 页（前 30 + 后 30），中间跳过
- 每页 50 行，共 3000 行
- 字号不小于 10pt（建议 11pt 等宽字体）
- 页眉：CodeCard 源代码
- 格式：PDF

### 源代码文件（已生成）

两个文件已经生成在项目里：

| 文件 | 行数 | 对应页数 |
|------|:--:|:--:|
| `docs/copyright/前30页-源代码.txt` | 1522 | ~30.4 页 |
| `docs/copyright/后30页-源代码.txt` | 1486 | ~29.7 页 |
| **合计** | 3008 | ~60 页 |

**前 30 页包含：** theme（颜色/字体/主题）→ types（类型定义）→ lib（工具函数）→ store（状态管理）→ navigation → screens（首页/课程/模块）

**后 30 页包含：** C++ 课程内容（面向对象/流/内存/条件/字符串）→ 动画注册表

### 转换成 PDF（3 步）

```
第 1 步   打开 WPS/Word → 新建空白文档
第 2 步   插入 → 文件 → 选"前30页-源代码.txt"
          设置格式：
            - 字体：Courier New，11pt
            - 行距：固定值 16 磅
            - 页边距：上 2.5cm / 下 2.5cm / 左 2cm / 右 2cm
            - 页眉：CodeCard 源代码
          导出 → PDF → "前30页-源代码.pdf"
第 3 步   同样操作"后30页-源代码.txt" → "后30页-源代码.pdf"
```

```

第 4 步   填表（新版申请表）：
          - 软件名称：CodeCard 学习软件 V1.0
          - 简称：CodeCard
          - 分类：应用软件
          - 开发完成日期：填实际完成日期
          - 首次发表日期：未发表
          - 硬件环境：Android 手机 / iPhone
          - 软件环境：Android 8.0+ / iOS 15+
          - 编程语言：TypeScript
          - 源代码量：13000 行（不含 node_modules）
          - 开发工具：VS Code + Expo SDK 55 + Claude Code（AI 辅助）
          - 功能描述：500-1300 字（重点！详述研发背景、核心架构、功能模块、技术实现）
          - 权利范围：全部权利
          - 著作权人：你的姓名 + 身份证号
第 5 步   手写抄录承诺段落（黑色中性笔，打印无效）
第 6 步   上传材料：源代码 PDF + 用户手册 PDF + AI 合规声明 PDF + 证明材料 + 身份证
第 7 步   提交 → 等受理通知书（3-5 个工作日）
第 8 步   审查（20-30 个工作日，涉 AI 项目可能更久）

### 找代理（推荐，涉 AI 项目首选）

```

## 常见被打回的原因

| 问题 | 规避 |
|------|------|
| 源代码页数不够 60 页 | 源码文档已凑够 3000 行 |
| 代码里有 `node_modules` 或第三方库 | 不要从 node_modules 取文件 |
| 用户手册截图太少或为 mockup | 至少 6 张真机运行截图，配技术说明 |
| 软件名称跟已有软著重名 | 加版本号"CodeCard 学习软件 V1.0" |
| PDF 不清晰/有水印 | 导出时选高质量 |
| **未提交 AI 合规声明** | 涉 AI 项目必须交，不交直接驳回 |
| **功能描述模板化/空泛** | 500-1300 字，写清楚研发背景 + 核心架构 + 技术实现 |
| **材料三重不一致** | 功能描述 ↔ 源代码 ↔ 用户手册必须完全对上 |
| **AI 生成内容未做实质性修改** | 需证明人工修改 ≥30%，附决策记录和设计文档 |
