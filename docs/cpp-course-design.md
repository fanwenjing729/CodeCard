### 4.1 流抽象 (6 卡)

> 从学生已知的 cin/cout 出发建立流概念。箭头方向 = 数据方向。cerr 的缓冲区差异是理解后面所有 I/O 缓冲行为的基础。

### 7.7 C++20 精选：if constexpr / 协程 / Modules (4 卡)

## 设计理念与分析

### 模块排列逻辑

8 个模块按"用 → 理解 → 造"三层递进：

```
01 基础         →  "用"          语法/控制流/函数/数组   ← 能用 C++ 写程序
02 进阶         →  "理解底层"    指针/内存四区/struct     ← 知道程序在内存中怎么跑
                + struct(→OOP)   递归                    + OOP 和算法的引桥
03 面向对象     →  "自己造类"    分文件/封装/继承/多态   ← 从使用者变成设计者
04 流与文件     →  "用标准库验证OOP"  fstream/stringstream    ← RAII 不再抽象
05 STL          →  "用标准库"    vector/map/algorithm    ← 生产力跃升
06 泛型         →  "造通用工具"  模板/特化/concept       ← 写可复用的代码
07 现代 C++     →  "跟上前沿"    lambda/移动/Ranges      ← C++11/17/20 关键特性
08 工程化       →  "交付工程"    CMake/GDB/Shell/规范    ← 从能写变成能交付
```

关键设计决策：

- **03 OOP 放在 04 流之前**：学生先懂 class、构造析构、RAII，再去看 ifstream/ofstream——知道构造函数打开文件、析构自动关闭是 RAII 在工作，而不是"反正这么写就对了"。从模糊的模仿变成有理论支撑的理解。
- **02 进阶插入 struct 和递归**：struct 是 OOP 的自然引桥（和 class 只有默认权限的区别），打破 7 个连续指针节点的密度。递归承接内存四区中"栈帧"的概念——让栈知识落到具体算法上。
- **分文件编写放在 03-oop 开头**：class 天然 `.h` + `.cpp`，学了分文件立刻用于写类，即学即用。
- **06 泛型紧接 05 STL**：STL 大量使用模板（vector`<int>`、map`<string,int>`），学生对模板"用"熟了，再学"怎么写"模板，顺势切入。
- **01.12 数组末尾加 vector 预览卡**：只播种不展开。学生知道前面有更好的选择，先踏实学数组理解底层。
- **07 智能指针已提前在 02-进阶讲**：动态内存管理是进阶模块的核心主题，学完堆/栈/new/delete 立刻引入 unique_ptr/shared_ptr 作为"不泄漏的保险"。07-modern 不再重复。

### 为什么网络编程和设计模式不在 C++ 课里？

| 主题     | 去处                | 理由                                                                   |
| -------- | ------------------- | ---------------------------------------------------------------------- |
| 网络编程 | `cn` 计算机网络课程 | socket 依赖 OS+计网基础，大一上太重。作为计网实践配套更合适            |
| 图形界面 | 不单独开模块        | C++ 标准库不含 GUI，所有方案（Qt/wxWidgets）都是第三方库 API，不是语言知识。工程化模块加一张概念卡交代清楚即可 |
| 设计模式 | 独立课程 `dp`       | 语言无关知识。OOP 学完后用 C++ + Python 对照讲，以后加其他语言也能复用 |
| 数据结构 | `ds` 数据结构课程   | C++ 学完 STL 后，用 C++ 实现一遍链表/树/图，比纸上写伪代码扎实         |

## 架构约定：防止导入链单点故障

### 问题

所有课程节点通过 3 层静态 `import` 串联：`courses/index.ts` → `cpp/index.ts` → 模块 index.ts → 节点文件。任意一个节点文件有语法错误，整个 `courses` 数组导出失败，所有课程页崩溃。

### 解决方案

课程数据通过 `src/lib/useCourses.ts` 访问层暴露，不再直接 import `courses`：

```
屏幕组件 → useCourse(id) / useCourses()  ←  React hook
工具函数 → getCourse(id) / getCourses()  ←  同步函数
          ↓
    src/lib/useCourses.ts（唯一访问点）
          ↓
    src/data/courses/index.ts（静态导出）
```

**规则**：
- 所有消费方通过 `@/lib/useCourses` 获取课程数据，**禁止**直接 `import { courses } from '@/data/courses'`
- 新增第二门课程时，只需改 `useCourses.ts` 内部实现——将静态 import 改为动态 `import()` + try-catch，对外 API 不变
- `validate.test.ts` 是编译期的安全网——提交前运行即可捕获断裂的 import 链

### 影响范围

数据层 `courses/index.ts` 仍是静态导出。当切换到多课程动态加载时，8 个消费方屏幕不需要任何改动——只改 `useCourses.ts` 一个文件。
