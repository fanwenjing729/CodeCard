| 材料 | 格式 | 说明 |

## ① 源代码文档制作

### 规则

- 60 页（前 30 + 后 30），中间跳过
- 每页 50 行，共 3000 行
- 字号不小于 10pt（建议 11pt 等宽字体）
- 页眉：CodeCard 源代码
- 格式：PDF

### 源代码文件（已生成）

两个文件已经生成在项目里：

| 文件 | 行数 | 对应页数 |
|------|:--:|:--:|
| `docs/copyright/前30页-源代码.txt` | 1522 | ~30.4 页 |
| `docs/copyright/后30页-源代码.txt` | 1486 | ~29.7 页 |
| **合计** | 3008 | ~60 页 |

**前 30 页包含：** theme（颜色/字体/主题）→ types（类型定义）→ lib（工具函数）→ store（状态管理）→ navigation → screens（首页/课程/模块）

**后 30 页包含：** C++ 课程内容（面向对象/流/内存/条件/字符串）→ 动画注册表

### 转换成 PDF（3 步）

```
第 1 步   打开 WPS/Word → 新建空白文档
第 2 步   插入 → 文件 → 选"前30页-源代码.txt"
          设置格式：
            - 字体：Courier New，11pt
            - 行距：固定值 16 磅
            - 页边距：上 2.5cm / 下 2.5cm / 左 2cm / 右 2cm
            - 页眉：CodeCard 源代码
          导出 → PDF → "前30页-源代码.pdf"
第 3 步   同样操作"后30页-源代码.txt" → "后30页-源代码.pdf"
```

## ③ 申请流程

### 自己办（电脑操作）

```
第 1 步   打开 https://www.ccopyright.com.cn
第 2 步   注册账号（手机号 + 身份证实名）
第 3 步   登录 → 版权登记 → 软件著作权登记 → 在线填报
第 4 步   填表：
          - 软件名称：CodeCard 学习软件 V1.0
          - 简称：CodeCard
          - 分类：应用软件
          - 开发完成日期：填当天
          - 首次发表日期：未发表
          - 硬件环境：Android 手机
          - 软件环境：Android 8.0+
          - 编程语言：TypeScript
          - 源代码量：13000 行
          - 开发工具：VS Code + Expo SDK 55
          - 权利范围：全部权利
          - 著作权人：你的姓名 + 身份证号
第 5 步   上传材料：源代码 PDF + 用户手册 PDF + 身份证
第 6 步   提交 → 等受理通知书（3-5 个工作日）
第 7 步   审查（20-30 个工作日）
第 8 步   公告 + 发证
```

### 找代理

```
淘宝搜"软件著作权申请" → 找销量高评价好的
→ 把材料发给对方（源代码 txt + 截图 + 身份证）
→ 代理填表 + 整理 PDF + 提交
→ 等 20-30 天出证
```

代理省掉第 4-5 步（填表和整理 PDF 格式），这两步是第一次做最容易被打回重审的环节。

## 常见被打回的原因

| 问题 | 规避 |
|------|------|
| 源代码页数不够 60 页 | 上面给的文件列表已经凑够 3000 行 |
| 代码里有 `node_modules` 或第三方库 | 不要从 node_modules 取文件 |
| 用户手册截图太少 | 至少 5 张，配文字 |
| 软件名称跟已有软著重名 | 加版本号"CodeCard 学习软件 V1.0" |
| PDF 不清晰/有水印 | 导出时选高质量 |
