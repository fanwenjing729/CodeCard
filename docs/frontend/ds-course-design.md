## 设计原则

- 每节开头给一个真实场景，回答"学这个有什么用"
- 代码卡全部可复制运行
- 覆盖考试经典题型（选择/填空/手算）
- 标注 🎯 高频面试点
- 砍掉大一用不上的学术内容，控制总量

## 整体结构

```
01 为什么要学(4) → 02 线性表(4) → 03 栈与队列(4) → 04 树(6) → 05 图(5) → 06 查找(4) → 07 排序(6) → 08 进阶(4)
```

**总计**：8 模块 / 37 节点 / ~248 卡

### 01-01 数据结构是什么（7 卡）

**核心目标**：知道"数据结构"四个字的准确含义，能区分逻辑结构和存储结构。

| 卡 | 类型 | 标题 | 设计要点 |
|:--:|------|------|----------|
| c1 | concept | 学数据结构到底学什么 | 从通讯录、地铁图、文件夹三个例子引出课程全貌。同一份数据不同组织方式 → 不同效率。一张图预览整门课会学的所有结构 |
| c2 | concept | 逻辑结构——数据之间的关系 | 四种：集合 / 线性 / 树形 / 图状。每种一个生活例子。408 考点 |
| c3 | concept | 存储结构——数据在内存里怎么放 | 顺序（相邻地址）/ 链式（指针串）/ 索引（目录）/ 散列（哈希算位置）。类比：宿舍同层 vs 寻宝线索 |
| c4 | concept | 同一逻辑两种存储——效率不同 | 线性表用数组 vs 链表，"在中间插入"谁快？悬念引导后续课程 |
| c5 | concept | 数据结构三要素 | 逻辑结构 + 存储结构 + 数据运算 = 一个数据结构。ADT = 屏蔽存储细节只暴露操作。408 高频 |
| c6 | practice | 判断：分辨逻辑与存储结构 | 4 道选择："线性表""顺序表""链表""栈"分别是逻辑还是存储。易混淆点刻意训练 |
| c7 | concept + practice | 算法是什么（附 408 考点） | 程序 = 数据结构 + 算法。算法 5 个特性（每个给反例：死循环无有穷性、random() 无确定性）。好的算法四个标准。2 道判断速测 |

> 01-02 独立节点已砍，算法特性压缩为一张卡。考试内容不缺，但不再占整节篇幅。

### 01-02 时间复杂度怎么算（9 卡）

**核心目标**：看到代码能算出 O(几)，达到考研难度。这是整个模块的**核心技能节点**。

| 卡 | 类型 | 标题 | 设计要点 |
|:--:|------|------|----------|
| c1 | concept | 怎么数操作次数 | 从单循环 `for(i=0;i<n;i++) sum+=a[i]` 入手。规则：忽略常数、忽略低阶、看增长最快的项。为什么"不数精确次数"——不同机器不同速度 |
| c2 | concept | 六种复杂度一张图 | O(1) 水平线 / O(log n) 几乎躺平 / O(n) 斜线 / O(n log n) 微弯 / O(n²) 抛物线 / O(2ⁿ) 爆炸。每条标注对应什么算法（二分/遍历/快排归并/冒泡/子集枚举） |
| c3 | concept | 加法规则与乘法规则 | 顺序执行取最大 O(max(f,g))，嵌套循环相乘 O(f×g)。举例：两个不同循环谁主导？嵌套循环为什么是平方？408 计算必用 |
| c4 | code | 实战：嵌套循环 | 经典 `for(i=0;i<n;i++) for(j=0;j<n;j++)` → O(n²)。变形 `for(j=0;j<i;j++)` 执行 n(n-1)/2 次 → 也是 O(n²)。408 爱考变形 |
| c5 | code | 实战：while 折半 | `while(low<=high) { mid=... }` 每次砍半 → O(log n)。关键：看到"每次缩小一半"就想到 log |
| c6 | concept | 最好 / 最坏 / 平均 | 顺序查找：O(1)/O(n)/O(n)。快排：O(n log n)/O(n²)/O(n log n)。大 O 默认最坏情况——保证任何输入都不崩。408 选择题高频 |
| c7 | concept | 递归复杂度怎么算 | 递归树法：每层工作量 × 层数。`T(n)=T(n/2)+1` → log n 层 → O(log n)。主方法常用结论：`2T(n/2)+n` → O(n log n)；`T(n-1)+1` → O(n)。**不推导公式，只记结论**——后续模块反复验证 |
| c8 | concept | O/Ω/Θ 有什么区别 | O 上限、Ω 下限、Θ 紧确。为什么平时只说 O——工程最关心最坏兜底。408 偶尔考区分，知道即可 |
| c9 | practice | 综合计算练习 | 4 道给代码选复杂度（单循环/嵌套/while折半/递归）+ 2 道三情况判断 + 1 道符号判断 |

### 01-03 空间复杂度与时空权衡（6 卡）

**核心目标**：理解"空间"也是代价，建立工程权衡意识。

| 卡 | 类型 | 标题 | 设计要点 |
|:--:|------|------|----------|
| c1 | concept | 空间复杂度是什么 | 算法额外占用的内存（不算输入数据）。递归调用栈深度 O(n)、归并排序临时数组 O(n)。408 考点：递归栈也算空间 |
| c2 | concept | 空间换时间 vs 时间换空间 | 哈希表 O(n) 空间换 O(1) 查找。嵌入式内存紧缺 → 时间换空间；服务器 → 空间换时间。**没有最优解，只有最适合的方案** |
| c3 | concept | 实战二选一 | 两个真实场景（手机 App vs 云端服务），让学生判断该用哪种策略，为什么 |
| c4 | concept | 常见数据结构的时空代价速查 | 一张表预览后续模块所有结构的时空复杂度（不要求记住，建立"每个结构有不同代价"的认知，后续模块逐个验证） |
| c5 | practice | 综合：分析完整代码 | 给"找数组中重复元素"的两种写法，分析各自时空复杂度并选择适用场景 |
| c6 | concept | 模块总结地图 | 回顾所有概念，一张图串联。预告下一模块：从最简单的线性结构开始 |

### 01-04 模块总复习 + 测验（4 卡）

**核心目标**：检验模块掌握程度，对标 408 真题。

| 卡 | 类型 | 标题 | 设计要点 |
|:--:|------|------|----------|
| c1 | practice | 408 真题 · 选择篇 | 5 道：逻辑/存储结构区分、算法特性判断、时间复杂度排序、最好最坏情况 |
| c2 | practice | 408 真题 · 计算篇 | 2 道：给代码算复杂度（含嵌套循环变形）、递归复杂度 |
| c3 | practice | 408 真题 · 综合篇 | 1 道：分析完整算法的时空复杂度并判断优劣 |

## 02 数据怎么排队——线性表（4 节点 / ~28 卡）

从 C++ 数组自然过渡，学生最熟悉的数据结构。

### 02-01 顺序表

**核心问题**：数组用得好好的，为什么要"顺序表"这个概念？

**卡片内容**：
- 从 `int arr[100]` 到动态扩容的 SeqList
- 插入删除为什么慢（元素要搬家）
- Size/Capacity 分离的思想
- 卡片类型：concept 3 + code 2 + practice 2

### 02-02 单链表

**核心问题**：不在内存里连续排队，怎么找到下一个？

**卡片内容**：
- 结点 = 数据 + 指针
- 头插/尾插/按位删除（画图 + 代码）
- "头指针不能丢"的血泪教训
- 卡片类型：concept 3 + code 3 + practice 2

### 02-03 双向链表与循环链表

**核心问题**：单向不够用怎么办？

**卡片内容**：
- 加个前驱指针的好处（删除不需要找前驱）
- 循环链表的判空技巧
- Linux 内核链表的思路（offsetof 宏）
- 卡片类型：concept 2 + code 3 + practice 1

### 02-04 我该用哪个 🎯

**核心问题**：面试官问"数组和链表有什么区别"怎么答？

**卡片内容**：

- CPU 缓存友好性
- 内存碎片
- 实际选择流程图
- 卡片类型：concept 3 + practice 3

## 03 先入后出与先入先出——栈与队列（4 节点 / ~26 卡）

从"撤销操作"和"排队叫号"切入，两个都是身边每天都在用的结构。

### 03-01 栈

**核心问题**：为什么 Ctrl+Z 能撤销？

**卡片内容**：
- LIFO 直觉 → push/pop/peek
- 顺序栈实现（top 指针）
- 链栈实现
- 🎯 面试：用数组实现栈
- 卡片类型：concept 2 + code 2 + practice 2

### 03-02 栈能干什么

**核心问题**：括号匹配、表达式计算——都和栈有关。

**卡片内容**：
- `{[()]}` 是否合法（遍历 + 栈判断）
- 中缀转后缀（运算符优先级）
- 后缀表达式求值
- 浏览器的前进后退原理
- 卡片类型：concept 3 + code 2 + practice 2

### 03-03 队列

**核心问题**：奶茶店叫号——先来先服务。

**卡片内容**：
- FIFO → enqueue/dequeue
- 顺序队列"假溢出"怎么来的
- front/rear 双指针
- 链队列
- 卡片类型：concept 2 + code 2 + practice 2

### 03-04 循环队列与双端队列

**核心问题**：队尾满了但队头有空位，浪费了。

**卡片内容**：
- 取模运算让队列转起来
- 队空 vs 队满的判定技巧（浪费一个位置 / 计数 / flag）
- deque 两端操作
- 卡片类型：concept 2 + code 2 + practice 1

## 04 从文件夹到决策树——树（6 节点 / ~40 卡）

树是第一个"不直观"的结构。用文件系统和 HTML DOM 建立熟悉感。

### 04-01 树长什么样

**核心问题**：文件夹套文件夹——你已经用了十年树。

**卡片内容**：
- 根/叶/父/子/兄弟/度/深度（用文件系统截图标注）
- 树 vs 二叉树的区别
- 树的三种表示法（双亲/孩子/孩子兄弟）
- 卡片类型：concept 3 + practice 1

### 04-02 二叉树

**核心问题**：最多两个分叉，为什么特别重要？

**卡片内容**：
- 5 条性质（手算验证每一条）
- 满二叉树 vs 完全二叉树（图对比）
- 顺序存储为什么浪费空间（斜树举例）
- 卡片类型：concept 3 + practice 3

### 04-03 遍历二叉树

**核心问题**：怎么"走遍"一棵树一个不落？

**卡片内容**：
- 前序/中序/后序——三个词概括区别（根在哪）
- 递归代码（3 行核心逻辑）
- 给两个序列画树（考试必考）
- 卡片类型：concept 2 + code 2 + practice 3

### 04-04 非递归遍历

**核心问题**：递归太深会爆栈，怎么办？

**卡片内容**：
- 用栈手动模拟递归（中序非递归）
- 层序遍历 = 用队列一层层扫
- 🎯 面试：层序遍历的 BFS 思想
- 卡片类型：concept 2 + code 3 + practice 1

### 04-05 二叉排序树（BST）

**核心问题**：查找像翻字典，比大小就知道往哪走。

**卡片内容**：
- BST 定义（左小右大）
- 查找/插入代码
- 删除的三种情况（叶/单子/双子 → 用前驱或后继替代）
- 中序遍历 = 升序输出
- 卡片类型：concept 3 + code 2 + practice 2

### 04-06 哈夫曼树

**核心问题**：怎么让文件压缩得更小？

**卡片内容**：
- 出现频率越高编码越短
- 哈夫曼构造过程（贪心选最小的两个合并）
- 前缀码——任何一个编码都不是另一个的前缀
- WPL 带权路径长度手算
- 卡片类型：concept 3 + code 1 + practice 2

> 砍掉：线索二叉树（大一用不上）、树与森林转换（太偏理论）。

## 05 地图导航与社交网络——图（5 节点 / ~34 卡）

用高德地图和朋友关系让图变具体。

### 05-01 图是什么

**核心问题**：微信好友关系、地铁线路——都是图。

**卡片内容**：
- 顶点/边/有向/无向/带权
- 度/入度/出度的含义
- 连通/连通分量/强连通
- 邻接矩阵画出来看
- 卡片类型：concept 4 + practice 1

### 05-02 图的存储

**核心问题**：邻接矩阵 vs 邻接表——空间换时间。

**卡片内容**：
- 稠密图用矩阵 O(n²)
- 稀疏图用表 O(n+e)
- 二维数组 vs 链式数组的实现差异
- 卡片类型：concept 2 + code 3 + practice 1

### 05-03 走遍整个图 🎯

**核心问题**：DFS 走迷宫、BFS 水波纹。

**卡片内容**：
- DFS 递归（一条路走到黑）
- BFS 队列（一层层扩散）
- visited 数组防重访
- 🎯 面试：DFS/BFS 的区别和应用场景
- 卡片类型：concept 2 + code 3 + practice 2

### 05-04 最小生成树

**核心问题**：如何用最短的电缆连接所有城市？

**卡片内容**：
- Prim（从一个点出发贪心选最短边）
- Kruskal（所有边排序 + 并查集防环）
- 两者对比：稀疏图用 Kruskal，稠密图用 Prim
- 卡片类型：concept 3 + code 2 + practice 1

### 05-05 最短路径 🎯

**核心问题**：高德怎么算最短路线？

**卡片内容**：
- Dijkstra——每次选当前最近的点，松弛它的邻居
- Floyd——三个 for 算出所有点对
- Dijkstra 不能处理负权边的原因
- 🎯 面试手写 Dijkstra
- 卡片类型：concept 3 + code 2 + practice 2

> 砍掉：拓扑排序/关键路径（太偏工程管理，大一听不懂）。

## 06 怎么找得更快——查找（4 节点 / ~28 卡）

从 Ctrl+F 到数据库索引。

### 06-01 顺序查找与二分查找 🎯

**核心问题**：翻字典——从第一页翻 vs 直接翻中间。

**卡片内容**：
- 顺序查找（哨兵优化，减少一次比较）
- 二分查找 while 循环实现
- mid 溢出 bug 及修复：`mid = left + (right-left)/2`
- 🎯 面试：二分查找的边界条件（left<=right 还是 left<right）
- 卡片类型：concept 2 + code 3 + practice 2

### 06-02 二叉排序树查找

**核心问题**：查找 + 插入 + 删除一体，动态维护有序性。

**卡片内容**：
- 动态查找表 vs 静态查找表
- BST 查找代码（小于往左大于往右）
- 删除后怎么保持 BST 性质
- ASL 平均查找长度手算
- 卡片类型：concept 2 + code 2 + practice 2

### 06-03 平衡二叉树（AVL）

**核心问题**：BST 插入顺序不好会退化成一根棍。

**卡片内容**：
- 平衡因子（左子树高 - 右子树高），绝对值 > 1 就失衡
- LL/RR/LR/RL 四种旋转（动画展示方向）
- 插入后最多转一次就恢复平衡
- 卡片类型：concept 3 + code 1 + practice 2

### 06-04 哈希表 🎯

**核心问题**：O(1) 查找——最快的查找方式。

**卡片内容**：
- 哈希函数（除留余数法）
- 冲突解决：开放定址（线性探测的聚集问题）/ 拉链法
- 装填因子与扩容时机
- Python dict / C++ unordered_map 的原理
- 🎯 高频面试
- 卡片类型：concept 3 + code 2 + practice 2

## 07 怎么排得更快——排序（6 节点 / ~42 卡）

排序是面试最爱考、考试必出题的内容。

### 07-01 排序怎么评价

**核心问题**：什么叫"排得好"？

**卡片内容**：
- 稳定性（相同 key 相对顺序不变，举例：成绩表先按姓名排再按分数排）
- 时间复杂度
- 空间复杂度（原地排序加分——不用额外数组）
- 卡片类型：concept 3 + practice 1

### 07-02 插入排序与希尔排序

**核心问题**：打扑克牌整牌——插到该在的位置。

**卡片内容**：
- 直接插入（摸牌插入已排好的手牌）
- 折半插入——查找位置用二分，但移动元素省不了
- 希尔排序——隔 gap 跳着插，gap 越来越小直到 1
- 为什么希尔比直接插入快：让元素大步逼近最终位置
- 卡片类型：concept 3 + code 2 + practice 2

### 07-03 冒泡排序与快速排序 🎯

**核心问题**：冒泡太慢 → 快排是最常用的排序。

**卡片内容**：
- 冒泡优化（某轮没交换就提前结束）
- 快排 Partition：选基准、挖坑填数、基准归位
- 分治递归
- 最坏 O(n²) 什么时候出现（已排序 + 选第一个做基准）
- 🎯 面试手写快排
- 卡片类型：concept 2 + code 3 + practice 2

### 07-04 选择排序与堆排序 🎯

**核心问题**：每次选最小/最大的放到前面。

**卡片内容**：
- 简单选择 O(n²) 但交换次数最少
- 堆——完全二叉树的数组表示（下标从 0 开始，left=2i+1）
- 建堆 O(n)（从最后一个非叶结点向下调整）
- 堆排序 O(n log n)（建堆 + n-1 次删堆顶）
- 🎯 面试：Top K 问题用堆
- 卡片类型：concept 3 + code 3 + practice 2

### 07-05 归并排序 🎯

**核心问题**：分成两半各自排好再合并。

**卡片内容**：
- 分治框架（递归到只剩一个元素自然有序）
- 合并两个有序数组（双指针）
- 稳定且 O(n log n) 但需要 O(n) 额外空间
- 🎯 面试：归并排序的稳定性有什么用？链表排序为什么用归并？
- 卡片类型：concept 2 + code 2 + practice 2

### 07-06 排序怎么选

**核心问题**：7 种排序一张表全部对比。

**卡片内容**：
- 一张大表：时间/空间/稳定性/适用场景
- 工程上为什么用快排（内排最快）+ 堆排（辅助）
- C++ std::sort 的实现策略（快排 + 堆排 + 插入混合）
- 卡片类型：concept 2 + practice 2

> 砍掉：基数排序（大一优先级低）。

## 08 进阶数据结构（4 节点 / ~24 卡）

学了这些，简历上的"熟悉数据结构"真的有底气。

### 08-01 并查集 🎯

**核心问题**：朋友圈——"你朋友的朋友也是你的朋友"。

**卡片内容**：
- 初始化（每个人是自己的老大）
- 查找（顺着找到最终老大）+ 路径压缩
- 合并（老大认老大）+ 按秩合并
- 判断两个人是否在同一圈
- 🎯 面试：判断图中是否有环
- 卡片类型：concept 3 + code 2 + practice 1

### 08-02 跳表

**核心问题**：链表太慢，加几层索引就起飞。

**卡片内容**：
- 多层索引思想（类似地铁快线/慢线）
- 插入时随机决定加几层（概率 1/2 逐层递减）
- 查找/插入/删除都是 O(log n)
- Redis Sorted Set 为什么用跳表而不是红黑树（实现简单 + 范围查询友好）
- 卡片类型：concept 3 + code 2 + practice 1

### 08-03 B 树与 B+ 树

**核心问题**：数据库索引为什么这么快？

**卡片内容**：
- 一个结点存多个 key（减少磁盘 IO 次数）
- m 阶 B 树的插入分裂、删除合并
- B+ 树叶结点用链表串起来（范围查询神器）
- MySQL InnoDB 默认索引结构就是 B+ 树
- 卡片类型：concept 4 + practice 1

### 08-04 字符串匹配

**核心问题**：Ctrl+F 的背后。

**卡片内容**：
- BF 暴力匹配（双重循环 O(mn)）
- KMP——利用已匹配信息避免主串指针回溯
- next 数组的直觉理解（前后缀最大相同长度）
- 卡片类型：concept 3 + code 3 + practice 1

> 砍掉：红黑树（太复杂，知道"平衡树"概念就够了）、外排序（大一不学）。
